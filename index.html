MMS Report Parser -ohjelman kehitysprosessi

Tämä dokumentaatio kuvaa MMS Report Parser -ohjelman kehitysprosessin vaihe vaiheelta. Ohjelman tarkoitus on lukea Movie Magic Scheduling -ohjelmasta tuotettuja PDF-raportteja, jäsentää niiden sisältö ja tallentaa tiedot Excel-taulukkoon.

PDF-raportit sisältävät tietoa käsikirjoituksen jokaisesta kohtauksesta. Jokainen kohtaus on erillisellä sivulla ja niissä kerrotaan mm. kohtausnumero, kuvauspäivämäärä, kohtauksen lyhyt kuvaus, kohtauksessa esiintyvät hahmot, avustajat, ajoneuvot jne. MMS Report Parser -ohjelma lukee raportin kaikki sivut ja tekee niistä Excel-taulukon jossa jokaiselle raportissa kerrotulle sivulle on oma rivi ja jokaiselle asialle oma sarake.

Toteutin ohjelman Pythonilla ja käytin apuna ChatGPT:tä koodin generointiin ja iterointiin. Tässä dokumentaatiossa käydään läpi käytetyt työkalut sekä kaikki vaiheet ohjelman rakentamisesta, virtuaaliympäristön luomisesta ja PDF-datan käsittelystä aina Excel-muotoiluun ja lopulliseen testaamiseen saakka. Jos haluat rakentaa samantyyppisen työkalun tai kehittää tätä eteenpäin, tästä dokumentaatiosta voi olla hyötyä!

Sivuston lopusta läydät linkin alkuperäiseen keskuteluun GPT:n kanssa, Movie Magic Scheduling -tiedoston käsikirjoituksen purusta, PDF-raportin, raportin layout-tiedoston, Excel-taulukon PDF-raportista ja itse ohjelman jonka toimivuutta voit testata antamani PDF-raportin avulla.

1. Ohjelmointikieli

Ohjelma on kirjoitettu Python-ohjelmointikielellä. Käytin versiota 3.12. Uusimmassa 3.13 versiossa minulla ilmeni ongelmia graafisen käyttöliittymän (Graphical User Interface, GUI) toiminnassa.

Pythonin voi ladata viralliselta sivustolta: https://www.python.org/downloads/release/python-3128/

2. Koodin tekstieditori

Koodin kirjoittamiseen käytän Visual Studio Code -tekstieditoria.

Editorin voi ladata viralliselta sivustolta: https://code.visualstudio.com

Uutta koodia voi aloittaa valitsemalla: File → New File → New File… “Python File Python”

3. Terminal-ohjelma (komentoriviohjelma)

Ohjelman rakentamiseen ja käyttämiseen käytän macOS:n Terminal-ohjelmaa. Terminal on komentorivipohjainen työkalu, jonka avulla hallitaan tietokonetta tekstikomennoilla graafisen käyttöliittymän sijaan.

Terminal ohjelma löytyy kansiosta:

Applications → Utilities → Terminal

4. Luo projektille hakemisto

Valitse haluamasi sijainti (esim. Documents) ja luo uusi kansio nimeltään esimerkiksi mms_report_parser.

Avaa komentorivi ja siirry kansioon kirjoittamalla komentoriville komento:

cd ~/Documents/mms_report_parser

Vaihtoehtoisesti: Ota Finderissä käyttöön View → Show Path Bar. Klikkaa ikkunan alareunassa olevaaa mms_report_parser -kansion kuvaketta hiiren oikealla napilla ja valitse “Open in Terminal”.

5. Virtuaaliympäristön luominen

Ohjelmia kannattaa rakentaa omassa virtuaaliympäristössään, jotta järjestelmän Python-asennus ja sen paketit eivät muutu ja mahdolliset konfliktit vältetään.

Virtuaali ympäristö luodaan syöttämällä komentoriville komento:

python3 -m venv venv

(Huom: jälkimmäinen "venv" on ympäristön nimi; käytän aina tätä nimeä, koska se helpottaa toistuvaa käyttöä.)

Ympäristö aktivoidaan komennolla:

source venv/bin/activate

Komentorivin alussa näkyy nyt ympäristön nimi, esim. (venv).

Virtuaali ympäristöstä pääsee tarvittaessa pois komennolla:

deactivate

8. Projektin alustaminen ja tiedostojen luonti Terminal-ohjelmassa

Kun olet navigoinut projektikansioon ja aktivoinut virtuaaliympäristön, luo tarvittavat tiedostot ja kansiot seuraamalla seuraavia ohjeita.

Luo requirements.txt-tiedosto:

touch requirements.txt

Avaa requirements.txt tekstieditorissa ja lisää sinne viitteet tarvittaviin kirjastoihin:

Pdfplumber
Openpyxl

Mitä kirjastot ovat?

Ohjelmointikirjastot (engl. libraries) ovat kokoelmia valmiiksi kirjoitettua koodia, joka tarjoaa tiettyjä toiminnallisuuksia. Pythonissa kirjastot asennetaan tyypillisesti pip-komennolla, ja ne helpottavat sovelluksen rakentamista, koska samoja toimintoja ei tarvitse toteuttaa itse alusta alkaen. Esimerkiksi pdfplumber-kirjasto sisältää valmiit työkalut PDF-tiedostojen lukemiseen ja openpyxl Excel-tiedostojen käsittelyyn.

Asenna kirjastot antamalla komentoriville seuraava käsky. Tarkista että olet virtuaaliympäristössä ennen käskyä:

pip install -r requirements.txt

Luo kansiorakenne ja python-tiedostot syöttämällä seuraavat käskyt:

mkdir modules
mkdir logs
touch main.py
touch modules/gui.py
touch modules/pdf_parser.py
touch modules/data_processor.py
touch modules/excel_writer.py
touch modules/logger_setup.py
touch logs/app.log

9. Koodin kirjoittaminen

Vastauksen lopuksi sain jokaiseen .py tiedostoon tulevan koodin. Kopion ja liitin ne sen vastaaviin .py-tiedostoihin Visual Studio Code -ohjelmassa.

Esimerkiksi kopion seuraavan koodin GPT:n vastauksesta

# modules/pdf_parser.py

import pdfplumber

def parse_pdf(pdf_path, logger):
    parsed_pages = []

    try:
        with pdfplumber.open(pdf_path) as pdf:
            for page_num, page in enumerate(pdf.pages, start=1):
                text = page.extract_text()
                if text:
                    lines = text.split('\n')
                    parsed_pages.append({
                        'page_number': page_num,
                        'lines': lines
                    })
                    logger.info(f"Parsed page {page_num}.")
                else:
                    logger.warning(f"No text found on page {page_num}.")
    except Exception as e:
        logger.error(f"Failed to parse PDF: {e}")
        return None

    return parsed_pages

Avasin pdf_parser.py tiedoston Visual Studio Code -ohjelmalla, liitin koodin sinne ja tallensin tiedoston.

10. Muutoksia koodiin

Ohjelma toimi lähes kuten piti. Huomasin kuitenkin joitain virheitä joita lähdin korjaamaan uusilla prompteilla:

Thank you! Two things. 1. Can we use tkinter for gui? 2. The pages were not converted to numbers as eights. Can we check that? Think it over first. Keep simple modular structure. No code needed yet. Thank you!

Promptin jälkeen GPT kertoi suunnitelman jonka jälkeen promptasin:

Thanks! Let's do these. Think it over first and then give me to compelte code for nessecary .py files. Do not change anything else but what we discussed.

Huomioita promptista: Do not change anything..., Tätä on hyvä käyttää, sillä joskus GPT keksii muuttaa ohjelmaa ihan omin luvin kummallisiin suuntiin.

Great! Thank you! Now the PAGES are numbers. However the excel cells are not formatted as eights. Could we fix that? I imagine it is simple change. Can you think it over first. No code needed. Thank you!"

Vastauksessa GPT antoi neuvoja siitä, mikä rivi missä .py tiedostassa tarvitsee muutosta. Tein korjauksen, ja Excel-muotoilu toimi halutusti:

modules/excel_writer.py
cell.number_format = '# ?/?' --> cell.number_format = '# ?/8'

Toinen prompti koski SHEET #-sarakkeen ongelmaa, jossa erotin merkin " : " siirtyi sarakkeeseen sheet-numeron mukana tiedostoon.

Great! Thank you! I made the change. Please remember that. There is a slight issue with the SHEET #. When it is parsed it takes with it the separator ":". For instance SHEET #: 8 is written as ": 8" in excel, when the correct would be "8". Could we fix that also. Think about it first. No code needed yt.

Promptissa pyysin että muistamaan aikaisemmin tehdyn muutoksen koodissa, sillä jos GPT kirjoittelee koodia uudelleen, niin ei palaa vanhaan versioon missä kahdeksasosat eivät toimi.

GPT:n vastauksessa sain heti korjatun koodin, vaikka sanoin että en tarvitse vielä koodia. Annettu koodi oli kuitenkin hyvä ja data siirtyi Excel-tiedostoon oikein.

Lopuksi pyysin lisäämään ominaisuuden, jossa käyttäjä voi valita, mihin ja millä nimellä Excel-tiedosto tallennetaan.

Thank you! Can we add a feature where we can determine where we save the excel file and how to name it? Think it over first. No code needed yet.

GPT:n vastauksen jälkeen promptasin.

Yes, we can do this. Let’s keep the modular structure. Do not add or change anything else. Think it over first and then give me the complete code for all the .py files that needs to be changed. Thank you!

GPT:n kirjoittama koodi toimi ja ohjelma oli näiltä osin nyt valmis.
